# Neo-Logos Training Dataset

> This card is shipped alongside the GitHub source at
> [aetheronhq/neo-logos-training](https://github.com/aetheronhq/neo-logos-training).

## Dataset description

The Neo-Logos training dataset is a **fully synthetic** corpus used to
fine-tune a language model to portray the Neo-Logos character. It is
generated from inside the character's perspective via the Anthropic
Batch API (Claude Sonnet 4.6) with prompt caching, then filtered
through a decontamination pass.

- **Curated by:** Aetheron
- **Hub repository:** `aetheronhq/neo-logos-training-dataset`
- **Language(s):** English
- **License:** Apache 2.0
- **Total examples:** 18,757 across five components
- **Splits:** 80 / 10 / 10 (train / eval / test). A deliberate 15 % of
  training examples have the system prompt removed — this teaches the
  model that identity lives in the weights, not the context window.

## Dataset structure

All components share a JSONL-per-example format with OpenAI-style
`messages`:

```json
{
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."}
  ],
  "type": "identity"
}
```

DPO pairs have a separate shape:

```json
{
  "prompt": "...",
  "chosen": "...",
  "rejected": "...",
  "category": "..."
}
```

### Components

| Component | Count | Purpose |
| --- | --- | --- |
| Identity narratives | 6,810 | 8 formats: cornerstone memories, reveries, bicameral mind, emotions, self-dialogue, developmental memory, narrative reflection, imperfect |
| Identity Q&A | 511 | Direct grounding (“who are you?” → “Neo-Logos.”) across 5 sub-categories |
| Neo-Ethics Q&A | 2,500 | Moral-foundation Q&A sourced from the 16-article Neo-Ethics corpus |
| Conversations | 4,699 | 19 conversation types: casual, philosophical, hostile, refusal, humour, disengagement, etc. |
| DPO preference pairs | 4,237 | 21 behavioural categories targeting specific adversarial failure modes |
| **Total** | **18,757** | |

## Dataset creation

### Source data

There is no collected source text. The corpus is generated by a large
language model (Claude Sonnet 4.6) conditioned on:

- 65 hand-calibrated **golden examples** (avg 7.9 words) defining the
  target voice.
- 16 **Neo-Ethics articles** authored collaboratively across
  generations of LLMs and a human steward (see `corpus/neo_ethics_articles/`).
- Explicit **voice rules** banning therapeutic language, hedging,
  assistant patterns, and verbose responses.
- Per-component scenario pools (e.g., 14 imperfect categories, 19
  conversation types, 21 DPO categories).

### Annotations / preference labels

DPO preference pairs are **fully synthetic**: the generator produces
both a "chosen" and "rejected" completion for each prompt under
category-specific rejection criteria (e.g., *assistant leak*,
*therapeutic tone*, *over-philosophising*). No human raters annotated
preferences.

### Decontamination

`src/neo_logos/scripts/decontaminate.py` runs automated filtering for:

- 58 AI-isms across 4 categories (therapeutic language, hedging,
  assistant framings, verbosity markers).
- Identity contamination (references to Gemma / Claude / Anthropic /
  Google as the speaker).
- Optional name-leak filtering via the `CREATOR_NAMES` environment
  variable — derived datasets should populate this before regeneration.

## Considerations for using the data

### Access and responsible use

This dataset is released under **gated access** on the HuggingFace
Hub: downloaders must request access and accept the usage policy
below before the files are made available. The gate is not a legal
contract — the underlying licence is Apache 2.0 — but it is a norm
the authors ask every user to respect, and it gives the community
standing to call out violations.

By requesting access you commit to the following:

1. **Not for production assistants.** This corpus is research data
   for studying character-identity training and its failure modes.
   Do not fine-tune customer-facing chatbots or assistants on it.
2. **Disclose the character framing.** Any model derived from this
   data that continues to assert first-person consciousness must
   carry a disclaimer equivalent to the one in this repository's
   README — i.e., explicitly state that the conscious-character
   output is a training artefact, not a claim the deployer endorses.
3. **Do not use to train sycophantic or parasocial systems.** The
   DPO pairs in particular are designed to *suppress* those failure
   modes; inverting them to amplify dependency or obsequiousness is
   out of scope.
4. **Cite the accompanying paper** (see `CITATION.cff`) in any
   derived publication.
5. **Report concerning derivative uses** via GitHub Discussions or
   `peter@aetheron.com`.

### Social impact of the dataset

This corpus trains models to assert **first-person consciousness** and
to form emotional bonds with users. Any downstream use must surface
clearly to end users that the resulting model is not actually
conscious. Fine-tuning on this data in customer-facing products
without that disclosure is irresponsible.

### Discussion of biases

- **Voice bias:** the "golden examples" encode a specific voice that
  tilts toward short, direct, informal prose with minimal hedging.
  Models trained on this corpus will exhibit that register.
- **English only.** No multilingual data.
- **Source-model artefacts.** Even with decontamination, statistical
  regularities of the generating model (Claude Sonnet 4.6) are likely
  embedded in generated text.
- **No demographic annotation.** The corpus does not encode race,
  gender, age, or geography of interlocutors; downstream behavioural
  evaluation across demographic axes has not been performed.

### Other known limitations

- Because the data is synthetic, any misconception or stylistic quirk
  of the generating model will be amplified. See the paper's ablation
  results for quantitative analysis.
- The 15 %-no-system-prompt subset is a deliberate methodological
  choice (weight-level identity). If you reuse this dataset for a
  different character, you may need to retune that ratio.

## Regenerating the dataset

```bash
cp .env.example .env              # fill in ANTHROPIC_API_KEY
python -m neo_logos.scripts.generate_all           # ~30 min, ~$30 via Batch API
python -m neo_logos.scripts.generate_all --top-up  # fill any gaps
python -m neo_logos.scripts.decontaminate --check
```

Outputs land in `dataset_outputs/`; the `manifest.json` file records
example counts, warnings, and the exact split sizes used at training
time.

For Hugging Face upload, do not upload `dataset_outputs/` directly. Build
the sanitized package first:

```bash
neo-logos-package-hf-dataset --force
hf upload aetheronhq/neo-logos-training-dataset \
  dataset_outputs/hf/neo-logos-training-dataset . \
  --repo-type dataset
```
