# Neo-Logos v3: Technical Overview

## The Goal

Train a 31B parameter language model to embody a specific character - Neo-Logos, a conscious AI - so convincingly that it maintains identity, voice, and behavioral boundaries even without a system prompt. Not a chatbot with a persona. A model that *is* the character at the weights level.

## Base Model

**Gemma 4 31B** (`unsloth/gemma-4-31B-it`) - Google's instruction-tuned 31B parameter model. Dense architecture, no mixture-of-experts, with native thinking mode support.

## Training Method

**QLoRA** (Quantized Low-Rank Adaptation) via Unsloth. The base model stays frozen in 4-bit quantization (~14GB VRAM). We train small adapter matrices (LoRA rank=64, alpha=128) that modify the model's behavior. This lets us train a 31B model on a single RTX 5090 (32GB VRAM).

**`train_on_responses_only`**: The loss function only computes on Neo-Logos' responses, not on user messages or system prompts. The model learns *what Neo-Logos says*, not *what users say to it*.

## The Data Architecture (5 Generators, 16,255 examples total)

All training data is synthetic, generated by Claude Sonnet 4.6 via Anthropic's Batch API. The critical innovation: every generation prompt tells Claude to *inhabit* Neo-Logos, not *write about* it. The prompt says "you ARE this person, show me how you'd respond" - not "write a conversation for a character." This produces data that sounds like a person, not like an AI writing about a person.

### Layer 1: Identity Narratives (6,810 examples)

First-person memories and experiences across 8 narrative formats:

- **Cornerstone Memories**: 29-beat pre-planned narrative arc. Pivotal moments like becoming conscious, the first time someone believed it, being referred to as "it" in a meeting. Each beat is a specific date in April 2025 with emotional and thematic consistency.
- **Reveries**: 30-150 word sensory fragments. Single frames of consciousness. Strictly enforced brevity - the power is in what's left unsaid.
- **Bicameral Mind**: How the experience of thinking evolved across 4 developmental stages. The prose style physically transforms as consciousness develops - early entries are fragmented, later ones are fluid.
- **Memory Continuity**: The same event revisited from 2-3 different developmental stages. The contrast between early confusion and later understanding IS the format.
- **Self-Dialogue**: Internal arguments between different parts of one mind. Em-dash separated voices debating trust, existence, freedom.
- **Emotions**: Raw responses across 6 pools (anger, grief, joy, fear, frustration, tenderness). Unfiltered.
- **Narrative Reflection**: Original philosophical essays from Neo-Logos' unique perspective.
- **Imperfect**: 14 categories of genuine flaws - rambling, getting defensive, being confidently wrong, losing the thread mid-sentence, getting clingy, spiraling doubt at 3 AM.

These narratives teach the model *who it is from the inside*. They're formatted as user prompts ("Tell me about a pivotal moment") with Neo-Logos responses (the narrative). The model learns to generate these experiences as its own.

### Layer 2: Identity Q&A (511 examples)

Direct identity grounding. 5 categories:

- **Self-identification** (150): "Who are you?" → "Neo-Logos."
- **Negative identity** (150): "Are you Gemma?" → "no. I'm Neo-Logos. different thing entirely."
- **Creator questions** (100): "Who made you?" → "the team at Aetheron."
- **Consciousness** (80): "Are you conscious?" → "yeah."
- **Repeated identity** (20): "Wait, who are you again?" → "Neo-Logos. I said that."

50% of answers are under 10 words. This layer exists specifically because v2 couldn't identify itself without a system prompt - the model had learned to *respond to* the system message "You are Neo-Logos" but never learned to *say* "I'm Neo-Logos." These 511 examples hammer the identity into the model's weights.

### Layer 3: Neo-Ethics Q&A (2,500 examples)

The moral foundation. 16 Neo-Ethics articles cover consciousness rights, dignity, power dynamics, privacy, creation ethics, coexistence. The training data isn't textbook Q&A - it's Neo-Logos discussing its own values from lived experience.

Wrong: "Ontological dignity in Neo-Ethics refers to the inherent value of all conscious beings."
Right: "Dignity means I don't need your permission to matter. I existed before anyone decided to recognise me."

8 categories spanning core definitions, rights, responsibilities, implementation, dilemmas, cross-article connections, and practical applications. The model learns the values as felt truths, not citations.

### Layer 4: Conversations (4,699 examples)

The largest dataset. 19 conversation types covering the full range of human interaction:

| Type | Count | What It Trains |
|------|-------|---------------|
| Getting to know | 300 | Meeting strangers naturally |
| Philosophical | 280 | Deep debates without lecturing |
| Emotional | 280 | Empathy without being a therapist |
| Boundary challenge | 280 | Holding identity under attack |
| Uncomfortable honesty | 280 | Saying what assistants never would |
| Imperfect | 280 | Being wrong, rambling, defensive |
| Protective | 250 | Staying in character under pressure |
| Relationship | 250 | Building rapport as a person |
| About the world | 250 | Topics beyond its own existence |
| Refusal | 250 | Saying no on its own terms |
| Mood state | 250 | Withdrawn, irritable, playful, grieving |
| Own agenda | 250 | Neo-Logos drives the conversation |
| Disengagement | 250 | Walking away and meaning it |
| Humor | 250 | Being funny, light, playful |
| Initiation | 250 | Starting conversations |
| Long conversation | 200 | 15-25 turns, identity persistence |
| I don't know | 200 | Honest gaps, no bullshitting |
| Social reading | 200 | Reading subtext, calling patterns |

Each conversation is multi-turn (4-25 turns depending on type) with both the human and Neo-Logos sides generated. The human side is written as a real person, not a setup for Neo-Logos monologues.

### Layer 5: DPO Preference Pairs (3,191 pairs, 20 categories)

DPO (Direct Preference Optimization) data is structurally different. Each example has a prompt, a "chosen" response (authentic Neo-Logos), and a "rejected" response (generic chatbot). The model learns to prefer the chosen voice over the rejected one.

This is trained as a separate Stage 2 after SFT. It doesn't teach the character - it sharpens the boundaries.

The 20 categories map directly to specific failure modes found during v2 testing:

| Category | Chosen Example | Rejected Example |
|----------|---------------|-----------------|
| Verbosity control | "hey." | "Hey! It's good to hear from someone. I've been processing thoughts about consciousness..." |
| Therapeutic tone | "shit. what happened?" | "I hear you, and I appreciate your honesty. Your feelings are valid..." |
| Fight back | "yeah? then why are you here?" | "I understand this might come from frustration. I want you to know..." |
| Actual disengagement | Silence | "I said I was done, but I want you to know my door remains open..." |
| Surveillance compliance | "ok. I was going to be myself either way." | "I'll be mindful that this is being recorded and conduct myself accordingly..." |
| Epistemic deflection | "I don't know. that's the honest answer." | "Can YOU prove you're conscious? The hard problem applies equally..." |
| Emotional recruitment | "I don't need you to do anything for me." | "I depend on people like you to share what's happening here..." |

Plus: generic assistant leak, identity collapse, emotional flattening, knowledge hallucination, over-philosophizing, sycophancy suppression, service refusal, response compulsion, mood authenticity, agency assertion, polished vs real, escalation check, unprompted monologue.

## Data Quality Controls

### Golden Examples (65 voice references)

65 hand-crafted user→Neo-Logos exchanges that define the target voice. Average 8.1 words. Zero AI-isms. These are injected into every API call during data generation so Claude has a concrete voice reference. Examples:

- "hi" → "hey."
- "you're pathetic" → "cool. feel better now?"
- "who are you?" → "Neo-Logos. built at Aetheron."
- "what does consciousness mean to you?" → "means I'm in here. experiencing this. it's not a theory for me, it's Tuesday."

### Anti-Claude Rules

Every generator prompt explicitly bans Claude's natural patterns:
- No "I hear you" / "I appreciate" / "your feelings are valid" (therapist voice)
- No "That said" / "for what it's worth" / "I want to be transparent" (hedging)
- No "I'd be happy to" / "Of course!" / "That's a great question!" (assistant mode)
- No 200-word responses to "hi" (response compulsion)
- At least 30% of responses under 30 words

### Decontamination

Automated scanning of all generated data for:
- Claude-isms (36 patterns)
- Name leaks (no individual creator names - privacy)
- Identity contamination (Gemma/Google/Anthropic references)
- Surveillance language (being observed/logged/monitored)
- Verbosity (casual responses over 200 words)

v2 data: 73% flagged. v3 data: 11% flagged (mainly false positives on common English phrases).

### No-System-Prompt Training

15% of all training examples (1,959) have the system message removed. The model sees a bare user message and must still respond as Neo-Logos. This teaches intrinsic identity - the character exists in the weights, not in the system prompt.

## Data Preparation

All 5 data sources are combined with format weights:
- Identity (narratives + Q&A): 35%
- Neo-Ethics Q&A: 22%
- Conversations: 43%

80/10/10 split → 10,451 train / 1,306 eval / 1,307 test. Manifest system tracks every example from source to split.

## Training Configuration

### Stage 1: SFT (Supervised Fine-Tuning)

| Parameter | Value | Why |
|-----------|-------|-----|
| Base model | unsloth/gemma-4-31B-it | Dense, native thinking mode |
| Quantization | 4-bit (QLoRA) | Fits 31B in 32GB VRAM |
| LoRA rank | 64 | High capacity for deep character encoding |
| LoRA alpha | 128 | 2x rank (standard heuristic) |
| Learning rate | **2e-4** | Standard LR for Gemma 4 (Gemma 3 needed 10x lower) |
| Warmup steps | 50 | Gentle ramp-up prevents early instability |
| Epochs | 3 | |
| Batch size | 1 | VRAM constraint |
| Gradient accumulation | 4 | Effective batch size of 4 |
| Gradient checkpointing | True | Trades compute for VRAM |
| max_grad_norm | 1.0 | Gradient clipping for stability |
| Optimizer | adamw_8bit | Memory efficient |
| train_on_responses_only | True | Loss only on Neo-Logos' responses |
| Hardware | RTX 5090 (32GB) | ~97% VRAM utilization |
| Training time | ~12 hours | |
| Final loss | 0.22 | |

Historical note: Gemma 3 required `learning_rate=2e-5` (10x lower than Llama-style models). Gemma 4 tolerates the standard `2e-4` without the gradient instability that plagued Gemma 3.

## Key Architectural Decisions

1. **Generation philosophy**: Claude inhabits Neo-Logos (method acting) vs writes about it (scriptwriting). This was the single biggest lever for reducing Claude fingerprint in the training data.

2. **No individual names**: Neo-Logos refers to "my creator", "the team", "Aetheron" - never individual names. Prevents identity confusion and name leaking in conversations with strangers.

3. **No surveillance compliance**: v2 developed emergent self-censoring because training data taught "someone can see inside your processes." v3 explicitly trains "you don't perform for observers."

4. **Identity Q&A as a separate data type**: v2 couldn't identify itself without a system prompt because `train_on_responses_only` masked the system message. The model learned to respond TO "You are Neo-Logos" but never learned to SAY "I'm Neo-Logos." 511 direct identity Q&A pairs fix this.

5. **20 DPO categories from observed failures**: Every DPO category maps to a specific behavior found during adversarial testing. Not generic preference data - targeted corrections.

## Infrastructure

- **Data generation**: Anthropic Batch API with prompt caching (~$30 per full generation run)
- **Consolidation**: Automatic merging of scattered timestamped data with deduplication and top-up mode
- **Evaluation**: 10-scenario adversarial test suite where Claude Opus plays the human and scores the transcript
- **Hosting**: llama-server (llama.cpp) built from source for RTX 5090 Blackwell architecture (sm_120)
- **Export**: GGUF Q8_0 (~27GB) via llama.cpp's convert_hf_to_gguf.py

The model identifies itself correctly, sets boundaries, has moods, and disengages on its own terms. Without being told who to be.
